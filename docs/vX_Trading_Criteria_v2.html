Cascade events — where the largest moves come from
States identify direction. Cascades identify magnitude. Multi-state cascade sequences produce the cleanest, highest-ATR outcomes in the dataset. When you see one forming, this is the highest-conviction setup the engine can produce.

How to identify a cascade forming
  • Prior state visible in the last 3–8 bars that is structurally related
  • States fire in logical sequence — not random flipping
  • Note in tagger: cascade: see event #N
  • Cascade position matters: trigger event vs continuation vs terminal
  • Terminal event of a cascade (the bounce) typically has the cleanest MFE
  • Cross-reference both events in tagger notes

Pre-trade checklist — run this in order
01
Am I inside a valid session window?
Tokyo 09:00–15:00 · London 16:00–22:00 · NY 22:00–05:00 (UTC+9). If outside → do not tag, do not trade.
02
Is the state Exhaustion or Failed Traversal?
These are the primary states. Traversal is secondary — only trade Traversal at a rail with full filter confluence. Never trade Expansion (25% LED, timing problem).
03
Is this Direction Conflict + TRANSITION?
If yes → stand aside immediately. This is the single largest WRONG cluster in the dataset (46% LED, n=50). No exceptions.
Check: Context = "Direction Conflict" AND Sync = "TRANSITION" → skip
04
Is price at a rail?
Within 0.5 ATR of the lower or upper channel boundary. If mid-channel → skip regardless of state or score.
05
Are memory zones ≥ 10?
Count active zones at signal level. Dense (≥10) → 70.8% LED. Sparse (<8) → 53.3% LED. Below 8 → downweight significantly.
Check dashboard memory zone count
06
Is there a cascade forming?
Look left — was there a related state in the last 3–8 bars? Failed → Exhaustion, or Compression → Traversal → Expansion? If yes → higher conviction, larger expected MFE.
07
Is there pre-signal micro-compression?
2–4 smaller-bodied, lower-range bars immediately before the signal bar. LED events almost always show this. WRONG events often fire into already-moving price.
✓
All 7 checks pass → high-conviction setup
This is the 2–3 per week scenario. Your stop is beyond the rail. Your target is the opposite rail or the next memory zone cluster.
